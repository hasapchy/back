# Чеклист: старые оприходования и накладные

1. После деплоя все существующие строки `wh_receipts` получают `is_legacy = true`, `is_simple = false`, `status = purchasing` — остаток по-прежнему считается по строкам прихода, накладные не обязательны.
2. Чтобы вести приход по накладным как в новом UI, создайте **новую** закупку (кнопка «Приход») с выключенным «Остаток как раньше»; перенос старых документов в новый режим без пересчёта остатков в этом релизе не автоматизирован.
3. Перед включением нового режима на боевых данных убедитесь, что склад разблокирован для инвентаризации, и сделайте резервную копию БД.
4. После обновления выполните `php artisan migrate` и при необходимости `php artisan db:seed --class=PermissionsSeeder`, если добавлялись новые права.
